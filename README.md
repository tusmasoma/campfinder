# campfinder
campfinder

## アーキテクチャ
レイヤードアーキテクチャ + DDD を採用

採用理由は、アプリケーションの構造を明確に分離して依存関係を管理しやすくすることで、複数人の開発でも品質を維持しやすくするためです。また、ビジネスの複雑さを効率的に扱えるドメイン中心の設計により、小規模から中規模のプロジェクトに適していたからです。
